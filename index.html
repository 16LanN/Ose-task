Git workflow

Что такое?

Git workflow - это правила, что помогают в разработке командных проектов в git, с возможностью контролировать версии git.

Зачем нужен?

Git workflow нужен для организованной и эффективной работы в команде. Он включает в себе возможность контролировать ветки, коммиты, версии кода - тем самым упрощая совместную работу.

Как работает?

Git workflow создает ветки, контролирует коммиты, сливает ветки и исправляет ошибки при слиянии веток.

Git fetch

Что такое?

Git fetch - это команда, что позволяет вам установить всё содержание других репозиторий, без внесений изменений в свой код.

Отличия от git pull.

Git fetch отличается от git pull тем, что при скачивании удалённого репозитория не трогает основной код, в то время как git pull скачивает и объединяет их.

Git rebase

Что такое?

Git rebase - это команда, что даёт нам возможность перенести коммиты с одной ветки на другую.

Чем отличвается от git merge.

Git rebase нам позволяет сохранить линейность в разработке, а git merge же лишь сливает последние коммиты веток, чтобы создать новый коммит, что был объединён из разных коммитов.
